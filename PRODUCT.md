# Product

## Register

product

## Users

Privacy-conscious users who need to manipulate PDFs (merge, split, compress, rotate, add signatures, extract pages, convert formats) without uploading files to a server. They trust their own device more than cloud services and prioritize keeping personal/sensitive documents offline.

## Product Purpose

A suite of 100% client-side PDF tools that run entirely in the browser with zero backend infrastructure. Users process files locally, files never leave the device, and there are no accounts, logins, or server-side logging. Success is measured by fast, intuitive tooling that builds confidence through transparency about how data is handled.

## Brand Personality

- **Tone:** Trustworthy, transparent, no-nonsense
- **3 words:** Private, Simple, Capable
- **Emotional goal:** Confidence—users should feel secure that their data stays theirs

## Anti-references

- Slick SaaS onboarding with dark patterns ("upgrade now")
- Signup walls or account creation requirements
- Vague privacy policies or hidden tracking
- Gratuitous animations that slow down workflows
- Overcomplicated UIs with features users don't need
- Heavy branding that competes with the actual tools

## Design Principles

1. **Privacy by default**: Visible, transparent proof that files never leave the device. Every page surface should reinforce this without being preachy.
2. **One tool, one job**: Each tool should do exactly one thing well. No feature creep or bundling unrelated options.
3. **Fast and predictable**: Minimal clicks to completion. Clear visual feedback at every step.
4. **Accessible to everyone**: WCAG AA compliance by default. No gatekeeping on ability.

## Accessibility & Inclusion

- WCAG 2.1 AA minimum
- Support for keyboard navigation and screen readers
- Sufficient color contrast (4.5:1 for body text, 3:1 for large text)
- No motion or flashing that could trigger vestibular issues
- Clear error messages and status feedback
